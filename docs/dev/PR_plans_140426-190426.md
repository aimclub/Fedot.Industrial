# План PR на 140426-190426

Стратегия строится не “по слоям кода”, а по снижению архитектурного риска. Сейчас у нас уже есть хороший фундамент,
поэтому следующий цикл лучше вести так, чтобы каждый PR уменьшал скрытую связность и одновременно повышал ценность для
benchmark/runtime.

## Планируемый порядок

1. **PR-1 Legacy Forecasting Boundary Cleanup**
   Самый дешёвый по риску и сразу уменьшает вероятность, что новый forecasting runtime всё ещё тайно зависит от старого
   industrial_*_strategy. После него база для всего остального буде чище.
2. PR-2 OKHS Primitive-Orchestration Extraction
   Самый важный архитектурный PR следующего цикла. OKHS всё ещё остаётся главным “остаточным монолитом”
3. PR-4 Repository and Search-Space Consolidation
   Тюнинг надо делать до полной нормализации vocabulary/defaults/search-space. Сначала выравниваем
   names/defaults/aliases, потом уже строим tuning orchestration поверх стабильного словаря.
4. PR-4 Explicit Stage-Tuning Contract
   После очистки + OKHS extraction + формализации нового репозитория моделей вводим stage-aware tuning:
   4.1. trajectory stage
   4.2. decomposition/rank stage
   4.3. forecast head stage
5. PR-5 Benchmark Stage Adoption Completion
   На этом шаге benchmark станет окончательным “зеркалом архитектуры”, а не просто местом запуска моделей.
6. PR-6 Neural Forecast Head Bridge
   Последний PR на данном этапе - он важный, но только после того, примитивы операций будут окончательно стабильны.

## Почему именно так
Сейчас главный риск не в том, что “чего-то не хватает”, а в том, что часть системы уже живёт в новой архитектуре, а
часть ещё может незаметно возвращать нас в старое легаси. Поэтому стратегия должна быть:

1. Убрать скрытую связность;
2. Стабилизировать source-of-truth;
3. Строить tuning/runtime orchestration;
4. Только после этого расширять систему дальше.

## Практический план цикла

1. Блок A: PR-1 + PR-2. Цель - сделать runtime и OKHS действительно shell-first.
2. Блок B: PR-4 + PR-3. Цель - зафиксировать vocabulary и поверх него ввести stage-aware tuning.
3. Блок C: PR-5 + PR-6. Цель - завершить benchmark adoption и подготовить neural bridge.
