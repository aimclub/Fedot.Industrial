## 0. Идея метода

Идея идет из диссертации и она сильнее, чем текущая реализация `kernel_ensemble`. В диссертации заложена достаточно
перспективная формулировка: **каждый генератор признаков рассматривается как способ построить собственное представление
временного ряда и соответствующее ядро; далее система должна адаптивно выбрать и скомбинировать эти ядра, одновременно
оптимизируя структуру композитной модели**. Это прямо связано с целью совместной оптимизации признакового пространства и
пространства структур моделей для классификации, восстановления зависимостей и прогнозирования временных рядов.

В коде эта идея реализована частично: `KernelEnsembler` строит несколько feature-generator pipelines, считает попарные
расстояния, прогоняет их через MKLpy и выбирает два наиболее полезных генератора. Но текущая реализация пока не является
полноценным модулем обучения ядер: нет строгого train/test kernel contract, нет гарантии PSD-ядер, нет регуляризации
сложности ядра из главы 3, нет полноценного fit/predict API, нет прямой поддержки регрессии и прогнозирования
в `kernel_ensemble`, а топологический генератор заявлен в node list, но выключен из baseline generators.

## 1. Краткий конспект идей из диссертации

Глава 2 формулирует метод автоматического построения признакового описания временного ряда. Основной принцип: вектор
признаков должен создаваться автоматически, должна быть возможность добавлять новые пространства признаков и объединять
их, а гиперпараметры feature extractors должны оптимизироваться вместе с гиперпараметрами предметных моделей. В
диссертации выделены четыре семейства признаковых преобразований: **статистические, частотные, топологические и
спектральные**.

Это можно записать как систему генераторов:

$G_j: X \rightarrow Z_j$

где $X$ — исходный временной ряд или набор рядов,
$G_j$ — генератор признаков,
$Z_j$ — признаковое пространство, созданное данным генератором.

Для задач классификации и регрессии $Z_j$​ используется как табличное представление для предметной модели. Для перехода
к ядрам логична формулировка:

$K_j(x_a, x_b) = \kappa_j(G_j(x_a), G_j(x_b))$

где каждый генератор признаков индуцирует собственное ядро.

### 1.2. Статистические признаки

Статистический генератор рассчитывает признаки по всей длине ряда и удаляет признаки с нулевой или почти нулевой
дисперсией. В методе различаются глобальные признаки — куртозис, эксцесс, показатель Херста, peak-to-peak amplitude,
автокорреляция — и локальные признаки по подпоследовательностям, например среднее, медиана, стандартное отклонение.
Оконные методы нужны, чтобы отслеживать изменение признаков во времени.
С инженерной точки зрения это самый дешевый и интерпретируемый генератор. Его стоит сохранить как бейзлайн по умолчанию
и как anchor-признаки для оценки прогнозируемости

Абстракция:

$G_{\text{stat}}(x) = [ s_1(x), \dots, s_p(x), s_1(x_{t:t+w}), \dots, s_p(x_{t:t+w}) ]$

### 1.3. Частотные признаки: wavelet/Fourier

Частотный генератор строит признаки на основе вейвлет-преобразования: используются не только коэффициенты аппроксимации,
но и положения пиков/впадин после преобразования; гиперпараметром выступает минимальный порог пика. Итоговый вектор
включает, например, число пиков выше порога и расстояния между пиками.

Абстракция:

$G_{\text{freq}}(
x) = [ \operatorname{coeffs}(W_\psi x), \operatorname{peaks}(W_\psi x; \tau), \operatorname{distances}(\operatorname{peaks}) ]$

Для развития ядрового подхода это естественный кандидат на семейство **frequency kernels**:

$K_{\text{wavelet}}(x_a, x_b) = \exp(-\gamma \|G_{\text{wavelet}}(x_a)-G_{\text{wavelet}}(x_b)\|^2)$

### 1.4. Топологические признаки

Топологический генератор трактует траекторную матрицу как облако точек, строит симплициальные комплексы через фильтрацию
Виеториса–Рипса или связности, затем извлекает топологические инварианты, включая числа Бетти. В диссертации отдельно
отмечено, что такой алгоритм устойчив к шуму и пригоден для генерации признаков без учителя.

Абстракция:

$T(x) \rightarrow P_x = \{p_i\}_{i=1}^m$ где T(x) — траекторная/Hankel-матрица, $P_x$​ — облако точек.

Далее строится фильтрация $\operatorname{VR}_\epsilon(P_x) \rightarrow [ \beta_0(\epsilon), \beta_1(\epsilon), \dots ]$
Топологический генератор особенно ценен для классификации промышленных временных рядов: он описывает форму/связность
динамики, а не только локальные значения. Но у него высокий риск вычислительной стоимости и сложности настройки.

### 1.5. Спектральные признаки и SSA/траекторная матрица

Спектральный генератор основан на преобразовании временного ряда в траекторную матрицу, SVD-разложении и группировке
компонент. Идея: ряд можно представить как сумму тренда, периодических компонент и шума; по собственным числам и
векторам можно выделять соответствующие компоненты.

Формально, для траекторной матрицы X:

$X_k = U \Sigma V^T$

Низкоранговое представление возвращает верхние k сингулярных векторов. В диссертации отдельно разобрана проблема
классического SVD: он вычислительно дорог, а аппроксимация чувствительна к спектру, особенно когда $\sigma_k$​ и
$\sigma_{k+1}$​ близки. Поэтому предлагается randomized low-rank approximation для траекторной матрицы.

Важная часть главы 2 — переход от нормы Фробениуса к спектральной норме для зашумленных данных с тяжелым хвостом
сингулярных значений. Предлагаемая гарантия:

$\|X - QQ^TX\|_2 \le (1+\epsilon)\|X-X_k\|_2$

Первые k сингулярных векторов интерпретируются как сигнал, а оставшийся хвост — как шум.

Глава 2 фактически задает **реестр признаковых генераторов**. С инженерной точки зрения каждый генератор должен стать
объектом с единым контрактом:

fit(X_train, y_train?) -> fitted generator
transform(X) -> FeatureBundle
kernel(X_left, X_right?) -> KernelBundle

С исследовательской точки зрения главная идея — не просто “сгенерировать больше признаков”, а построить разные *

## 2. Глава 3 диссертации: конспект идей и алгоритмов

### 2.1. Adaptive feature description via MKL

Для адаптивного подбора числа моделей признакового пространства в главе 3 выбраны ядровые методы. Ядро неявно задает
представление данных, но выбор оптимального ядра сложен; классический перебор ядер и гиперпараметров вычислительно
дорог, поэтому предлагается использовать MKL — multiple kernel learning.

Базовая формализация:

$K = \{k(x_i, x_j)\}_{i,j=1}^n$

Через feature map:

$\phi: \mathbb{R}^p \rightarrow \mathbb{R}^q$
$k(x_i, x_j) = \phi(x_i)^T\phi(x_j)$

В диссертационной логике генераторы из главы 2 становятся такими $\phi_j$j​, а их ядра — базовыми ядрами $K_j$​.

Комбинированное ядро:

$K_\alpha = \sum_{j=1}^{m} \alpha_j K_j, \quad \alpha_j \ge 0, \quad \sum_j \alpha_j = 1$

Если добавить sparsity, то ненулевые α автоматически выбирают число активных генераторов.

### 2.3. Регуляризация сложности ядра

Одна из наиболее ценных идей главы 3 — учитывать не только alignment/separability, но и сложность результирующего ядра.
Ранг ядра может использоваться как мера сложности, но требует диагонализации. Поэтому предлагается спектральная мера:

$C(K)= \frac{\|K\|_T}{\|K\|_F} = \frac{\sum_i K_{i,i}} {\sqrt{\sum_{i,j}K_{i,j}^2}}, \quad 1 \le C(K) \le \sqrt{r(K)}$

Эта мера позволяет аппроксимировать сложность ядра дешевле, чем через явный ранг, и включать ее как регуляризатор в
функцию потерь.

Практическая целевая функция может выглядеть так:

$\max_{\alpha} \operatorname{Align}(K_\alpha, K_y) - \lambda_C C(K_\alpha) - \lambda_S \|\alpha\|_1 - \lambda_D \sum_{i<
j}\alpha_i\alpha_j \operatorname{sim}(K_i, K_j)$
Здесь:

- $\operatorname{Align}(K_\alpha, K_y)$ — alignment с target kernel;
- $C(K_\alpha)$ — регуляризация сложности ядра;
- $\|\alpha\|_1$или group sparsity — выбор числа активных генераторов;
- penalty по сходству ядер — защита от выбора взаимно избыточных генераторов.

### 2.4. Эволюционная оптимизация структуры

Метод главы 3 состоит из двух этапов: сначала эволюционный алгоритм ищет структуру композитной модели с использованием
отбора, скрещивания и мутаций; затем гиперпараметры найденной структуры донастраиваются, например, байесовской
оптимизацией. В тексте отмечен важный риск: одновременный поиск топологии и гиперпараметров может ухудшать сходимость.

Для кода это означает: **kernel selection лучше вынести в отдельный устойчивый слой**, а не смешивать все сразу в одном
огромном пространстве FEDOT. Сначала — выбрать/взвесить генераторы, затем — отдать компактное пространство в AutoML.

### 2.5. Forecasting-модификация

В диссертации для прогнозирования использована модификация: генераторы `[topological, statistical]` строят kernel
matrices; далее считаются попарные расстояния, применяется spectral clustering; если тестовый ряд попадает в кластер,
применяется SSA + autoregressive model, иначе ARIMA.

Это можно развить в более общий **kernel-gated forecasting**:

g(x)=arg⁡max⁡cp(c∣Kα(x,Xtrain)),g(x) = \arg\max_c p(c \mid K_{\alpha}(x, X_{\text{train}})),g(x)=argcmax​p(c∣Kα​(
x,Xtrain​)), y^t+h=fg(x),h(x),\hat{y}_{t+h} = f_{g(x),h}(x),y^​t+h​=fg(x),h​(x),

где кластер/гейт выбирает forecasting head: SSA+AR, OKHS, DLinear, ARIMA, N-HiTS, FEDOT-generated pipeline.

## Предлагаемое развитие идеи диссертации

### 3.1. Новая формулировка: генератор признаков как kernel-producing operator

Каждый генератор должен иметь два режима:

Feature mode:
X -> Z

Kernel mode:
X_left, X_right -> K

Для генератора $G_j$​:

$Z_j = G_j(X)$
$K_j = \kappa_j(Z_j, Z_j)$

Для тестовой выборки:

$K_j^{test,train} = \kappa_j(G_j(X_{test}), G_j(X_{train}))$

Это снимает текущую проблему, где считается только матрица расстояний на трейне.

3.2. Единый объект результата

Нужен объект:

KernelBundle:
name: str
train_kernel: np.ndarray
test_kernel: np.ndarray | None
train_features: np.ndarray | None
test_features: np.ndarray | None
is_psd: bool
psd_correction: str | None
complexity: dict
diagnostics: dict

Такой объект позволит:

- логировать качество генераторов;
- проверять PSD;
- считать C(K);
- строить train/test kernels;
- кэшировать тяжелые генераторы;
- объяснять, почему выбран конкретный генератор.

### 6.3. Adaptive sparse MKL

Базовая задача:

$\max_{\alpha \in \Delta} A(K_\alpha, K_y) - \lambda_C C(K_\alpha) - \lambda_R R(\alpha) - \lambda_D D(
K_1,\dots,K_m;\alpha)$

где:

$K_\alpha = \sum_j \alpha_j K_j$​.

Для классификации:

$K_y(i,j) = \mathbb{1}[y_i=y_j]$

Для регресcии:

$K_y(i,j)=\exp(-\gamma_y(y_i-y_j)^2)$

Для прогнозирования:

$K_y(i,j)= \exp(-\gamma_y \cdot d(y_{i,t:t+h}, y_{j,t:t+h})^2)$

или horizon-specific $K_y^{(h)}$​.

### 3.4. Class-specific, horizon-specific и cluster-specific weights

Текущая реализация выбирает лучший генератор по классу. Это хорошая идея, но реализация должна быть более строгой:

- **class-specific MKL**:

$\alpha^{(c)} = \arg\max_\alpha A(K_\alpha, K_{y=c})$

- **horizon-specific MKL** для прогнозирования:

$\alpha^{(h)} = \arg\max_\alpha A(K_\alpha, K_y^{(h)})$

- **cluster-specific MKL**:

$\alpha^{(r)} = \arg\max_\alpha A(K_\alpha, K_y \mid x \in cluster_r)$

Это позволит понять, что, например, wavelet хорошо работает для сезонных классов, topology — для геометрически
устойчивых режимов, SSA — для трендовых рядов, foundation embeddings — для сложных heterogeneous datasets.

### 3.5. Forecastability-aware prior

Cпектральная энтропия интерпретируется как мера сложности: низкая spectral entropy означает концентрированную энергию на
немногих частотах и высокую прогнозируемость, высокая — шумоподобность и низкую прогнозируемость.

Это можно использовать как prior для генераторов:

low spectral entropy -> wavelet/Fourier/SSA prior ↑
high entropy + nonlinear -> topology/shapelet/foundation embeddings ↑
trend-heavy -> SSA/DLinear/OKHS prior ↑
high multivariate coupling -> iTransformer/MOMENT/OKHS/multiview kernels ↑
